# ADR-035: Package Rename — sovereign_memory to audittrace

**Status:** Accepted
**Date:** 2026-04-17
**Deciders:** Luis Filipe de Sousa
**Related:** ADR-014 (Python package structure), ADR-026 (multi-user identity)

## Context

The project originated as "sovereign-memory-server" — a local-first
sovereign AI memory proxy. Over three months it evolved into
AuditTrace-AI: a full audit-trail-first memory augmentation platform
with OAuth2 identity, per-user RLS, EU AI Act Article 12 compliance,
and a three-model inference topology.

The Python package remained `sovereign_memory`, all environment
variables used the `SOVEREIGN_` prefix, Docker containers were named
`sovereign-*`, the database was `sovereign_ai`, and the Keycloak realm
was `sovereign-ai`. This naming mismatch between the repository
identity (`AuditTrace-AI`) and the runtime identity
(`sovereign_memory`) created confusion for anyone encountering the
codebase and would be cemented into the Helm chart if not addressed
before the k8s deployment (the next milestone).

## Decision

Hard rename of every runtime reference with no backward-compatibility
shims. The rename is a single coordinated change across Python code,
Docker infrastructure, environment variables, shell scripts, database
roles, and Keycloak realm configuration.

### Mapping

| Old | New |
|-----|-----|
| Python package `sovereign_memory` | `audittrace` |
| Env prefix `SOVEREIGN_*` | `AUDITTRACE_*` |
| Container `sovereign-memory-server` | `audittrace-server` |
| Database `sovereign_ai` | `audittrace` |
| DB roles `sovereign` / `sovereign_app` | `audittrace` / `audittrace_app` |
| Keycloak realm `sovereign-ai` | `audittrace` |
| Docker network `sovereign-ai-net` | `audittrace-net` |
| JWT audience `sovereign-memory-server` | `audittrace-server` |
| OAuth2 scopes `sovereign-ai:*` | `audittrace:*` |
| Redis key prefixes `sovereign:*` | `audittrace:*` |

### What was NOT renamed

- **OTel span attributes** (`sovereign.component`, `sovereign.operation`,
  `sovereign.operation.duration`) — these are observability metric names
  baked into Langfuse dashboards and Grafana queries. Renaming them would
  break existing dashboards without user-visible benefit. They can be
  migrated independently when the k8s observability stack is rebuilt.
- **ADR documents** — historical records kept as-is. This ADR documents
  the rename for anyone reading older ADRs that reference the old names.
- **TLS certificate filenames** (`certs/sovereign.pem`) — these are local
  dev artefacts generated by mkcert, not brand-carrying names.

## Consequences

**What becomes easier:**

- Repository name, package name, container names, and env vars are
  aligned. A new contributor reading `AUDITTRACE_LLAMA_URL` immediately
  knows which project it belongs to.
- The Helm chart (next milestone) will use `audittrace-*` naming from
  day one — no confusing mix of old and new names in the k8s manifests.
- Brand consistency across allaboutdata.eu, the GitHub repository, and
  the running system.

**What becomes harder:**

- Existing `.env` files are invalid. Operators must regenerate via
  `scripts/setup-secrets.sh` or manually rename all variables.
- Docker volumes from the old stack (`postgres_data`, `chroma_data`,
  `sovereign_redis_data`) are orphaned. Fresh deployment required —
  no in-place migration of the existing database.
- Keycloak realm name changed — existing tokens and Device Flow
  configurations are invalid. Re-provisioning via
  `scripts/setup-human-user.sh` required.
- Any external system referencing the old env vars, container names,
  or scope names must be updated (OpenCode config, Bruno collections,
  CI pipelines).

**Deployment:**

A deployment runbook (`docs/guides/deployment-runbook.md`) ships
alongside this ADR with step-by-step instructions for a fresh
deployment from zero to running.
