## Context

The memory server requires:
1. **Testability** - All database dependencies must be mockable for unit tests
2. **Observability** - Structured logging for debugging and production monitoring
3. **Zero file I/O** - Logs to stdout only (Kubernetes/container-native)
4. **Configurable verbosity** - DEBUG level shows input/output, INFO level for production

Traditional logging patterns create coupling between application logic and logging infrastructure, making unit testing difficult.

## Decision

### 1. Factory Pattern for Database Dependencies

We implement the **Factory Pattern** with **Dependency Injection** to allow seamless swapping of database implementations:

```python
class ChromaDBFactory(ABC):
    @abstractmethod
    def get_client(self) -> ChromaDBClient: pass

class SQLiteChromaDBFactory(ChromaDBFactory):  # Phase 0
    def get_client(self) -> ChromaDBClient:
        return chromadb.PersistentClient(path=self.persist_dir)

class HTTPChromaDBFactory(ChromaDBFactory):  # Phase 1+
    def get_client(self) -> ChromaDBClient:
        return chromadb.HttpClient(host=host, port=port)

class MockChromaDBFactory(ChromaDBFactory):  # Testing
    def get_client(self) -> ChromaDBClient:
        return MockChromaDB()
```

**Dependency Container**:
```python
container = DependencyContainer()
container.register_factory("chromadb", SQLiteChromaDBFactory(persist_dir="./data"))
client = container.get_instance("chromadb")
```

### 2. Logging as an Aspect (FluentPython Pattern)

Inspired by Luciano Ramalho's *FluentPython*, we use **logging decorators** to implement logging as an aspect:

```python
@log_call(logger=logger)
def store(self, project: str, source: str, content: str) -> str:
    # Business logic
    pass
```

**Features**:
- **Input logging**: Logs method arguments at DEBUG level
- **Output logging**: Logs return values at DEBUG level
- **Duration logging**: Logs execution time at DEBUG level
- **Error logging**: Automatically catches and logs exceptions
- **Conditional**: Only logs when `log_level == DEBUG`

### 3. Log Configuration

All logs go to **stdout only** (no file I/O):

```python
def setup_logging(
    level: str = "INFO",
    structured: bool = False,
    include_input_output: bool = False,
) -> None:
    # Configure handler for stdout
    handler = logging.StreamHandler(sys.stdout)
    logger.addHandler(handler)
```

**Configuration Options**:
- `level`: DEBUG, INFO, WARNING, ERROR, CRITICAL
- `structured`: JSON formatting for log aggregation
- `include_input_output`: Only when DEBUG

### 3. Dependency Injection in Routes

FastAPI's `Depends()` integrates with our DI container:

```python
async def get_memory_service() -> MemoryService:
    """Dependency: get memory service (mock in tests)."""
    return MockMemoryService()  # In tests, ChromaMemoryService in production

@router.post("/context")
async def get_context(
    request: ContextRequest,
    memory_service: MemoryService = Depends(get_memory_service),
):
    context = memory_service.retrieve(request.query)
    return ContextResponse(context=context)
```

## Consequences

### Positive

1. **Unit tests without databases**
   - All tests run in memory with mocks
   - No integration test setup required
   - Achieve 90%+ code coverage easily

2. **Zero coupling to logging infrastructure**
   - Decorators handle logging automatically
   - No logging statements in business logic
   - DEBUG vs INFO controlled by config

3. **Seamless database migration**
   - SQLite → PostgreSQL: just change factory
   - Tests automatically use mocks
   - No code changes in routes

4. **Container-native logging**
   - All logs to stdout
   - Kubernetes can collect logs
   - No file permissions issues

5. **Observability**
   - DEBUG mode shows full request/response
   - Duration tracking for performance monitoring
   - Structured JSON for log aggregation (ELK, Splunk)

### Negative

1. **Initial overhead**
   - Need to create factory classes
   - Need to create mock implementations
   - More files than monolithic approach

2. **Dependency injection complexity**
   - Must register all dependencies
   - Must use `Depends()` in all routes
   - Learning curve for new developers

3. **Debug decorator limitations**
   - Must decorate each method individually
   - Can add performance overhead in DEBUG mode
   - Sensitive data in logs must be filtered manually

### Neutral

- **Code organization**: More files, but clearer separation of concerns
- **Test setup**: More fixtures, but more powerful isolation

## Implementation Details

### Factory Pattern

**Location**: `src/sovereign_memory/db/factory.py`

**Classes**:
- `ChromaDBFactory` - Abstract base class
- `SQLiteChromaDBFactory` - Embedded mode (Phase 0)
- `HTTPChromaDBFactory` - Server mode (Phase 1+)
- `MockChromaDBFactory` - Testing (Phase 0)

**Usage**:
```python
# Production
factory = SQLiteChromaDBFactory(persist_dir="./chroma_data")
client = factory.get_client()

# Testing
factory = MockChromaDBFactory()
client = factory.get_client()
```

### Logging Decorator

**Location**: `src/sovereign_memory/logging_config.py`

**Decorator**:
```python
@log_call(
    logger=logger,
    include_input=True,
    include_output=True,
    include_duration=True,
)
def my_method(self, arg1, arg2):
    pass
```

**Log Output (DEBUG mode)**:
```
2026-04-09 20:00:00 - INFO - INPUT store(project='test', source='opencode', content='...')
2026-04-09 20:00:00 - INFO - OUTPUT store: 'uuid-1234'
2026-04-09 20:00:00 - INFO - DURATION store: 0.0012s
```

### Dependency Container

**Location**: `src/sovereign_memory/dependencies.py`

**Functions**:

**Coverage Goal**: 90%+

## Examples

### Unit Test with Mocked DB

```python
def test_context_endpoint_with_mock_memory(client):
    """Test context endpoint with mocked memory service."""
    response = client.post("/context", json={
        "query": "test query",
        "project": "test-project",
        "limit": 10,
    })

    assert response.status_code == 200
    data = response.json()
    assert "context" in data
    # No real database required!
```

### Phase 1

- Logs to stdout, DEBUG in dev
