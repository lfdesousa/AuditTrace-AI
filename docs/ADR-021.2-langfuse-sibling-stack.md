# ADR-021.2: Langfuse as Sibling Compose Stack

**Status:** Accepted
**Date:** 2026-04-10
**Deciders:** Luis Filipe de Sousa
**Related:** ADR-014.4 (Observability), ADR-020 (PostgreSQL)

## Context

Langfuse v3 requires PostgreSQL + ClickHouse + Redis + MinIO. Embedding these
in our compose file would create tight coupling with upstream Langfuse releases
and mix observability infrastructure with application infrastructure.

## Decision

Run Langfuse as a **separate sibling compose stack** using the official
`langfuse/langfuse` docker-compose.yml, connected to our services via a shared
external Docker network (`sovereign-ai-net`).

### Architecture

```
sovereign-ai-stack/                  langfuse/ (sibling directory)
├── docker-compose.yml               ├── docker-compose.yml (upstream)
│   ├── memory-server ──────────────>│   ├── langfuse-web
│   ├── postgres (sovereign data)    │   ├── langfuse-worker
│   ├── chromadb                     │   ├── postgres (traces only)
│   └── traefik                      │   ├── clickhouse
│                                    │   ├── redis
└── sovereign-ai-net (shared) ──────>│   └── minio
                                     └── docker-compose.override.yml
```

### Setup

```bash
./scripts/setup-langfuse.sh
# Creates sovereign-ai-net network
# Downloads official Langfuse compose
# Writes override to attach langfuse-web to shared network
```

### OTLP Endpoint

Memory-server sends traces to Langfuse v3's native OTLP ingest:
```
SOVEREIGN_OTLP_ENDPOINT=http://langfuse-web:3000/api/public/otel
```

This works because both stacks share `sovereign-ai-net`.

## Why Separate Stacks

1. **Blast radius** — Langfuse schema migrations cannot lock sovereign audit tables
2. **Lifecycle independence** — upgrade Langfuse without touching sovereign services
3. **Backup divergence** — audit data has long retention; trace data has short TTL
4. **Sovereignty story** — "your data in this Postgres, observability in that one"
5. **Upstream maintenance** — delegate Langfuse glue to the official compose

## Consequences

### Positive
- Zero coupling between sovereign data and observability data
- Langfuse upgrades are `docker compose pull && up -d` in the sibling directory
- Clean procurement narrative for regulated clients
- No custom Langfuse Dockerfile or configuration to maintain

### Negative
- Two compose stacks to manage (mitigated by setup script)
- Shared network must be created before either stack starts

### Neutral
- Phase 4 (Kubernetes) merges both into a single Helm chart with separate namespaces
