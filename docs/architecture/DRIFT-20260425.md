One-pass audit of `docs/architecture/` versus the deployed state, run
**before** the M1/Vault chart restructuring so any pre-existing drift

rather than surfacing as noise mid-implementation.

M1 PR alongside the Vault changes. Items not touched by M1 surfaces
(Container, Deployment view, Trust-boundary) are queued for later

## Drift findings

### D1 — chat-model identity is stale

`workspace.dsl:14` reads
`Qwen3.5-35B-A3B Q4_K_M on :11435 — chat + tool-loop reasoning`
and `workspace.dsl:204` repeats the same string in the deployment
node.

**Reality (2026-04-24):** chat model on `:11435` is
**`Qwen 3.6-27B-Q4_K_M`**. The 35B-A3B variant was swapped out the
day a Q8_0 variant of 27B failed to load (HIP graph-compile stall on
gfx1151; documented in `project_chat_model_q4_27b` memory).

`product-and-dependencies.md:129` repeats the same drift in prose:
"Qwen 3.6-35B-A3B for chat" — should be 27B.

**Fix in M1 PR:** Container and Deployment views both touched by M1
(Vault box added). Update the chat-server description in the same
edit pass. **Cost:** two strings.

### D2 — ADR numbering for upcoming gaps is out of date

`product-and-dependencies.md:66` says
"Addressed by roadmap Phase 1.2 (ADR-039), target 2026-05-16" for
IdP federation, and `:121` says
"Addressed by roadmap Phase 1.1 (ADR-040), target 2026-05-16" for
Vault. The same numbering shows up in the consolidated gap table
at `:206–211`.

**Reality:** the next free ADR numbers are **ADR-043** (Vault, this
milestone) and **ADR-044** (IdP federation, next milestone). ADRs
039 and 040 were tentative reservations; they have not been written
and the roadmap supersedes them.

**Fix in M1 PR:** the Vault row in the gap table is the row M1
closes. Update the ADR number to ADR-043 and the status to
"shipping in M1". The IdP row is updated to ADR-044 in the same
pass to keep the table consistent. The Trust-boundary diagram for
M1 needs the same updated ADR pointer.

### D3 — Keycloak misclassified as a Bitnami subchart

`product-and-dependencies.md:63` reads
"Default (dev) | Bitnami Keycloak subchart bundled in the Helm
umbrella."

**Reality:** Keycloak is templated directly in
`charts/audittrace/templates/keycloak/` (Deployment + Service +
Secret + VirtualService + AuthorizationPolicy). It is **not** a
subchart. `Chart.yaml:16–24` lists only `postgresql` and `redis`
as Bitnami subchart dependencies; MinIO and Keycloak ship as
hand-rolled templates.

**Fix in M1 PR:** This is a Container-view-touched line because M1
is moving the Keycloak admin password to Vault. Correct the prose
in the same edit pass: "Bundled Keycloak in the Helm umbrella
(hand-templated, not a Bitnami subchart)."

### D4 — `Q4_K_M` claim does not match `:11435` reality everywhere

The DSL chat-server line at `:14` says `Qwen3.5-35B-A3B Q4_K_M`.
The current `:11435` model is `Qwen 3.6-27B-Q4_K_M`. The Q4_K_M
suffix is correct; the dimensions are wrong (see D1).

This is the same fix as D1; called out separately because a future
M4 (model-tier) milestone will revisit this label and adding a
`tier=Q4|Q8|FP16` annotation column. **Not in M1 scope.**
