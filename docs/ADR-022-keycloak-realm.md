# ADR-022: Keycloak Realm Configuration

**Status:** Accepted
**Date:** 2026-04-10
**Deciders:** Luis Filipe de Sousa
**Related:** ADR-023 (JWT validation), ADR-020 (PostgreSQL)

## Context

Phase 1 delivered the infrastructure (Docker Compose, PostgreSQL, TLS). Phase 2
adds authentication so no endpoint is publicly accessible without a valid JWT.

## Decision

Use **Keycloak 24** as the identity provider with **private_key_jwt** client
authentication (RFC 7521/7523).

### Realm: `sovereign-ai`

Imported automatically via `realm-sovereign-ai.json` on first Keycloak startup.

### Client Scopes

| Scope | Purpose |
|---|---|
| `sovereign-ai:query` | Chat completions + session save |
| `sovereign-ai:context` | Memory context retrieval |
| `sovereign-ai:audit` | Interaction audit trail |
| `sovereign-ai:index` | Memory indexing |
| `sovereign-ai:admin` | Health + metrics endpoints |

### Clients

| Client ID | Scopes | Use Case |
|---|---|---|
| `opencode-agent` | `query` | OpenCode coding agent |
| `continue-agent` | `query`, `context` | Continue IDE extension |
| `roocode-agent` | `query` | Roo Code agent |
| `inject-memory` | `context`, `index` | Memory injection tool |
| `admin-client` | `admin`, `audit` | Administration |

All clients use `private_key_jwt` — no shared secrets. RSA-2048 key pairs
generated by `scripts/generate-client-keys.sh`.

### Why private_key_jwt

- No shared secrets — private key never leaves the agent host
- Non-repudiation — each assertion is cryptographically signed
- Revocation — remove public key from Keycloak to revoke
- RFC 7521/7523 — standard, auditable, regulator-friendly

### Infrastructure

- Keycloak runs in the same Docker Compose stack
- Uses a separate `keycloak` database in the same PostgreSQL instance
- Accessible via Traefik at `/realms/*` and `/admin/*`
- Admin credentials via `KEYCLOAK_ADMIN_PASSWORD` environment variable

## Consequences

### Positive
- All endpoints protected by JWT with scope enforcement
- Reproducible realm via JSON import — no manual configuration
- Each agent has its own identity and audit trail
- Clean revocation model via public key removal

### Negative
- Keycloak adds ~500MB to the Docker image footprint
- Key management overhead (one RSA key pair per agent)

### Neutral
- `auth_enabled=false` bypasses all authentication (development mode)
- Health endpoint (`/health`) remains unauthenticated for probes
